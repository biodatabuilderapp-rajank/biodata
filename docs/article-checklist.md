# BiodataBuilder — Article Definition of Done (DoD)

> An article is **NOT ready to publish** until ALL applicable checks pass.

---

## 1. Search Intent

- [ ] Primary keyword identified
- [ ] Secondary keywords identified
- [ ] Search intent fully satisfied (informational / navigational / transactional)
- [ ] First paragraph immediately answers the query

---

## 2. Content Quality

- [ ] Original content (not rephrased from a single source)
- [ ] Better than top 5 competitors on depth and usefulness
- [ ] Covers both beginner and advanced questions
- [ ] No obvious missing topic
- [ ] No AI repetition or filler phrases
- [ ] Word count meets or exceeds the top-ranking competitor
- [ ] Content reviewed and verified by a real person (not purely AI-generated)

---

## 3. Structure

- [ ] TL;DR / Key Takeaway near the top
- [ ] Table of Contents (for articles > 1000 words)
- [ ] Single, clear `H1`
- [ ] Proper `H2` section hierarchy
- [ ] Proper `H3` sub-section hierarchy (where needed)
- [ ] Conclusion section

---

## 4. Required Sections

Every article should contain the following **where applicable**:

- [ ] Definition — What is this topic?
- [ ] Why it matters — Why should the reader care?
- [ ] Step-by-step guide
- [ ] Real example (Indian names, Indian context)
- [ ] Table — structured comparison or data
- [ ] Checklist — actionable list for the reader
- [ ] Pro Tips
- [ ] Common Mistakes to Avoid
- [ ] Comparison (e.g., with alternatives or older practices)
- [ ] FAQ section
- [ ] Next Steps / What to do now

---

## 5. Visual Content

- [ ] Hero image present
- [ ] At least 2 supporting images
- [ ] At least 1 infographic
- [ ] All images have meaningful `alt` text (not just filenames)
- [ ] Images are optimised in size (WebP preferred, < 200KB each)
- [ ] Images placed near the relevant text, not just at the top
- [ ] Infographic is visually shareable (not just a decorative element)

---

## 6. User Experience

- [ ] Easy to scan — reader can understand the article from headings alone
- [ ] Short paragraphs (3–4 lines max)
- [ ] Bullet lists used where appropriate
- [ ] Tables used for structured data
- [ ] Mobile-friendly layout verified
- [ ] No content hidden behind interaction only (accordions hiding key info)

---

## 7. EEAT (Experience, Expertise, Authoritativeness, Trustworthiness)

- [ ] Practical, actionable advice — not just theory
- [ ] Indian cultural context respected (caste, religion, regional customs where relevant)
- [ ] Human examples with realistic Indian names and scenarios
- [ ] Trustworthy, verifiable information
- [ ] Demonstrates clear expertise on the topic

---

## 8. SEO

- [ ] Primary keyword in the `H1` title
- [ ] Primary keyword in the URL slug
- [ ] Primary keyword in the first paragraph
- [ ] Primary keyword in the conclusion
- [ ] Related/secondary keywords naturally included throughout
- [ ] Meta title written (< 60 characters)
- [ ] Meta description written (140–160 characters)
- [ ] Canonical URL set correctly in metadata
- [ ] `datePublished` / `dateModified` shown on the page (trust signal for readers and Google)
- [ ] Internal links added (minimum — see Section 11)
- [ ] External references added where relevant (authoritative sources)

---

## 9. Rich Results / Schema

- [ ] FAQ section present and eligible for FAQ schema
- [ ] Snippet-friendly lists (numbered or bulleted, concise)
- [ ] Snippet-friendly tables (clear headers, scannable)
- [ ] Definition paragraph (clear, concise — eligible for definition snippet)
- [ ] How-to steps (eligible for HowTo schema where applicable)
- [ ] `BreadcrumbList` JSON-LD schema added (auto-handled by `ArticleWrapper`)
- [ ] `Article` JSON-LD schema: `headline`, `datePublished`, `dateModified`, `author`

---

## 10. Product Integration

- [ ] CTA near the top (above the fold or just below the intro)
- [ ] CTA in the middle of the article
- [ ] CTA at the end / conclusion
- [ ] BiodataBuilder naturally recommended in context
- [ ] PDF download mentioned
- [ ] WhatsApp sharing mentioned
- [ ] Templates mentioned

---

## 11. Internal Linking

- [ ] Links to parent article (if this is a child/sub-topic)
- [ ] Links to child articles (if this is a parent topic)
- [ ] Links to related articles
- [ ] Link to `/create` (the builder page)
- [ ] Minimum **10 internal links** across the article

---

## 12. Freshness & Maintenance

- [ ] "Last updated" date is accurate and shown to the reader
- [ ] Article scheduled for review every **6 months** (add to a reminder or tracking sheet)
- [ ] Seasonal content (wedding seasons, regional trends) flagged for timely updates

---

## 13. Accessibility

- [ ] Sufficient colour contrast on all text (WCAG AA minimum)
- [ ] All images have meaningful `alt` text
- [ ] No critical information locked behind interaction (e.g., collapsed accordions)
- [ ] Headings are not skipped (H1 → H2 → H3, never H1 → H3)

---

## 14. Final Review — The Hard Questions

Ask yourself:

- [ ] Is this the best article on this topic on Google right now?
- [ ] Does it answer every question a reader might have?
- [ ] Would I bookmark this article?
- [ ] Would I share this with my family member who is making a biodata?
- [ ] **Can an AI generate this article in 5 minutes?** — If YES, add more unique value before publishing.

---

## Publishing Rule

The article must score **95%+** on all five dimensions before publishing:

| Dimension              | Target |
|------------------------|--------|
| Usefulness             | 95%+   |
| Readability            | 95%+   |
| Topical Completeness   | 95%+   |
| Conversion Opportunity | 95%+   |
| Uniqueness             | 95%+   |
